# Warm Flow Jimmer ORM adapter

This module adds a Jimmer-backed ORM adapter for Warm Flow. It keeps the Warm Flow runtime contract mutable, while persisting through Jimmer immutable SQL models and typed query DSL classes.

## Modules

| Module | Purpose |
| --- | --- |
| `warm-flow-jimmer-core` | Jimmer models, Warm Flow entity DTOs, converters, DAO implementations, and PostgreSQL smoke tests. |
| `warm-flow-jimmer-sb-starter` | Spring Boot auto-configuration that registers Warm Flow beans and wires the application's `JSqlClient`. |
| `warm-flow-jimmer-sb3-starter` | Spring Boot 3 dependency-aligned wrapper around the base starter. |
| `warm-flow-jimmer-sb4-starter` | Spring Boot 4 dependency-aligned wrapper around the base starter; runtime depends on upstream Jimmer Spring Boot 4 compatibility. |

## Maven

Use one Warm Flow ORM starter in the application. For PostgreSQL, include the Jimmer starter module and a PostgreSQL JDBC driver if your application does not already provide one.

```xml
<dependency>
    <groupId>org.dromara.warm</groupId>
    <artifactId>warm-flow-jimmer-sb-starter</artifactId>
    <version>${warm-flow.version}</version>
</dependency>

<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <scope>runtime</scope>
</dependency>
```

For Spring Boot 3 applications, use `warm-flow-jimmer-sb3-starter`. For Spring Boot 4 applications, use `warm-flow-jimmer-sb4-starter` only after validating the selected Jimmer release with Boot 4 in your runtime. The version-specific modules are dependency wrappers; auto-configuration metadata is supplied by `warm-flow-jimmer-sb-starter`.

## PostgreSQL configuration

Create the Warm Flow schema with `sql/postgresql/postgresql-warm-flow-all.sql`, then configure the datasource and Jimmer dialect:

```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://localhost:5432/warm_flow
    username: warm_flow
    password: ${WARM_FLOW_DB_PASSWORD}

jimmer:
  dialect: org.babyfish.jimmer.sql.dialect.PostgresDialect
  database-validation-mode: ERROR
  show-sql: false
  pretty-sql: false

warm-flow:
  enabled: true
```

`PostgresDialect` is required for production PostgreSQL deployments. The adapter maps PostgreSQL quoted/reserved columns explicitly, including `"version"` on definition/form/node tables and `"type"` on the user table.

## Runtime model

Jimmer immutable interfaces cannot directly implement Warm Flow's mutable entity interfaces. The adapter therefore uses two representations:

1. `org.dromara.warm.flow.orm.model.*Model` Jimmer immutable models for persistence.
2. `org.dromara.warm.flow.orm.entity.Flow*` mutable Warm Flow DTOs for the engine and service layer.

`JimmerConverter` translates between them. DTO-only fields that are not present in the Warm Flow DDL, such as permission lists and joined display names, remain on the mutable DTOs and are intentionally not persisted by Jimmer.

## Operational notes

- IDs are still generated by Warm Flow before persistence; the Jimmer models treat `id` as an assigned key.
- Tenant and logical-delete defaults follow the existing Warm Flow adapter behavior.
- User-provided `orderBy` strings are not passed to SQL directly. The adapter accepts only simple property/column names and a whitelist of supported order fields.
- The starter requires a Spring-managed `JSqlClient`. If it is missing, startup fails fast with a `BeanCreationException`.

## Verification

Run the Jimmer module tests:

```bash
mvn -q -pl warm-flow-orm/warm-flow-jimmer -am test
```

The PostgreSQL integration smoke is opt-in and creates a temporary schema that is dropped during cleanup:

```bash
mvn -q -pl warm-flow-orm/warm-flow-jimmer/warm-flow-jimmer-core \
  -Dtest=JimmerPostgresIntegrationTest \
  -Dwarm.jimmer.it.postgres.url=jdbc:postgresql://<postgres-host>:5432/<database> \
  -Dwarm.jimmer.it.postgres.user=<user> \
  -Dwarm.jimmer.it.postgres.password='<password>' \
  test
```

You can also provide the same values with environment variables:

- `WARM_JIMMER_IT_POSTGRES_URL`
- `WARM_JIMMER_IT_POSTGRES_USER`
- `WARM_JIMMER_IT_POSTGRES_PASSWORD`
